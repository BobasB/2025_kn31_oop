# Лабораторна робота №6: Віртуальні середовища Python

## Огляд

Ця лабораторна робота присвячена вивченню віртуальних середовищ Python та інструментів управління залежностями. Віртуальні середовища дозволяють ізолювати залежності різних проектів, уникаючи конфліктів між версіями пакетів.

## Мета роботи

- Навчитися працювати з різними інструментами управління віртуальними середовищами
- Зрозуміти переваги та недоліки кожного підходу
- Навчитися управляти залежностями проекту
- Створювати відтворювані середовища розробки

## Інструменти, що розглядаються

### 1. **venv** - Вбудований інструмент Python
Стандартний модуль Python для створення віртуальних середовищ. Простий у використанні, але має обмежений функціонал.

### 2. **Pipenv** - Сучасний менеджер пакетів
Об'єднує pip та virtualenv, додаючи файл Pipfile для декларативного управління залежностями.

### 3. **Poetry** - Повнофункціональний менеджер проектів
Сучасний інструмент, що надає комплексне рішення для управління залежностями, збірки та публікації пакетів.

## Структура проекту

```
6_lab/
├── app.py              # Базовий приклад застосунку
├── anime.py            # Flask-застосунок з API
├── 1_venv/             # Приклад роботи з venv
│   ├── requirements.txt
│   └── requirements-dev.txt
├── 2_pipenv/           # Приклад роботи з pipenv
│   └── Pipfile
└── 3_poetry/           # Приклад роботи з poetry
    ├── pyproject.toml
    ├── mkdocs.yml
    └── docs/           # Документація (ви тут!)
```

## Використані бібліотеки

У цій роботі використовуються наступні бібліотеки:

- **httpx** - сучасний HTTP-клієнт для Python
- **requests** - класична бібліотека для HTTP-запитів
- **Flask** - мікрофреймворк для веб-додатків
- **jikanpy-v4** - клієнт для MyAnimeList API
- **flake8** - інструмент для перевірки стилю коду (dev-залежність)
- **mkdocs** - генератор статичних сайтів для документації

## Змінні середовища

У проекті використовуються змінні середовища для конфігурації:

- `URL_TEST` - URL для тестування HTTP-запитів
- `TEST` - тестова змінна для демонстрації роботи з env

## Запуск документації

Для перегляду цієї документації локально:

```bash
cd 6_lab/3_poetry
mkdocs serve
```

Документація буде доступна за адресою: `http://127.0.0.1:8000`
